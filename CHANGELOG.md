# MirrorNovel生成器 - 更新日志

> 项目从零到一的全记录

## v1.0.0 初始版本 (2026-05-18)

### 项目初始化
- 搭建完整项目结构：`server/`（Node.js + Express 后端）、`client/`（Vue3 + Vite 前台）、`admin/`（Vue3 + Vite 管理后台）
- MongoDB 数据库设计，数据库名 `xiaoshuoplus`
- 配置硅基流动 DeepSeek-V4-Flash AI 模型

### 后端基础能力
- **用户系统**：邮箱注册（163 邮箱验证码）、JWT 登录、个人信息管理
- **数据库模型**：User（用户）、Novel（小说含章节）、VerificationCode（验证码）
- **小说生成**：7 种预置小说类型（玄幻修仙、都市异能、科幻未来、武侠江湖、悬疑推理、言情恋爱、历史穿越），每种含关键字、大纲模板、AI 推荐用词
- **流式生成**：基于 SSE（Server-Sent Events）的实时流式文本生成，用户可见 AI 逐字输出
- **书架系统**：小说 CRUD，生成中/已暂停/已完成/出错状态管理
- **导出功能**：单本/批量导出为 ZIP 压缩包，内含整本 TXT + 分章节多份 TXT

### 前台页面
- **底部 Tab 导航**：生成小说 / 书架 / 我的
- **生成页**：类型选择 → 主角设定 → 世界观设定 → 字数选择 → 实时流式展示生成内容
- **书架页**：书籍列表、状态标记、继续生成/删除操作
- **小说详情页**：章节列表展示、继续生成
- **个人中心**：登录/注册（邮箱验证码）、创作统计、昵称修改

### 管理后台
- 独立项目 `admin/`，端口 5174
- **数据大屏**：注册用户数、小说总数、Token 统计、生成完成数等
- **用户管理**：搜索、编辑、禁用/启用、追加 Token
- **小说管理**：按用户/状态/标题筛选
- **模型管理**：修改系统 API 地址/模型名/Key

## v1.1.0 (2026-05-18)

### Token 计费系统
- 新增 `tokens` 字段（total/used），新用户注册赠送 1000 Token
- Token 消耗按输出字符实时折算（中文 1.5 token/字，英文 0.3 token/字）
- **Token 水球**：个人页动态水球动画展示余额
- **假支付宝充值**：15 元/百万 Token，弹窗模拟扫码支付
- 生成前余额校验，余额不足时停止生成
- Token 不足时弹出引导充值提示

### 公告弹窗
- 登录后弹出公告：加 QQ 群送 Token
- 关闭后不再显示（存入用户表）

### 续写功能
- 底部新增「小说续写」Tab
- 支持粘贴小说内容或上传 .txt 文件
- 续写后的内容追加到原书章节（不创建新书）
- 生成设置弹窗（字数 + 备注）

### 章节操作
- 小说详情页每章节支持：编辑、删除、继续生成
- 未完成章节展示「继续生成」和「生成设置」按钮
- 删除章节自动重排后续章节号

### 骨架优化
- 修复 MongoDB 认证连接问题
- 修复暂停不保存已生成数据的问题（使用 `finalSave` 确保数据持久化）
- 每 5 秒自动保存生成进度
- 修复生成中提示样式错乱（`.generating-indicator` CSS 选择器问题）

## v1.2.0 (2026-05-19)

### 生成模式优化
- 「生成一本」/「生成一章」单选模式
- 一本模式最大支持 100 万字
- 一章模式支持 500~8000 字
- 切换模式自动修正字数预设

### 大纲系统
- Novel 模型新增 `outline` 字段
- 整本生成时展示大纲输入框，用户可填写创作大纲
- 未填写时由 AI 自动生成大纲
- 书架每本书新增「📋 大纲」按钮，弹窗编辑
- 续写/继续生成时自动携带大纲作为上下文

### 续写质量提升
- **蒸馏提纯**：`distillChapters()` 从全部章节提取关键剧情脉络
- 续写不再只依赖上一章，而是提供完整剧情上下文（每章最多 3000 字，总上限 10000 字）
- 提示词要求 AI 注意伏笔回收和全局连贯性

### 去 AI 味系统（基于 oh-story-claudecode）
- `server/config/deslop.js`：完整禁用词表（32 个一级禁用词 + 13 种句式模板）
- 三遍去 AI 法：去泛化 → 去书面化 → 回人味
- 改写范例库（情绪外化、场景描写、结尾改写等）
- 生成时自动注入去 AI 味系统提示词
- 详情页每章节新增「✨ 去AI味」按钮
- `POST /api/novel/deslop` 后端去 AI 味接口

### 管理后台增强
- 管理后台拆分为独立项目 `admin/`（端口 5174）
- 用户管理新增「进群赠送」按钮，赠送 5000 Token
- 用户模型新增 `groupRewardClaimed` 字段标记领取状态
- 服务器部署于腾讯云硅谷节点（49.51.51.253）

### 权限与安全
- AI 模型配置仅管理员可见（`role === 'admin'`）
- 管理员启动时自动创建（admin@xiaoshuo.com / admin888）
- 用户账号禁用机制
- MongoDB 远程连接支持（端口 27017 开放）

### 优化与修复
- 修复 chapterBuffer 作用域问题导致暂停不保存
- 修复 `$push: undefined` 导致 MongoDB 更新静默失败
- 修复 Token 字数重复累加
- 修复 `continue-chapter` 路由创建新章节报"章节不存在"
- 修复续写传到已有书时 `typeName is not defined`
- 修复 Mongoose 子文档变更追踪问题（`markModified`）
- 底部间距增加 24px 防止 QQ 群提示遮挡按钮
- 全文"番茄" → "镜像" 品牌改名
- 默认提供商文案优化
